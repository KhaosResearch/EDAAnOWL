Síntesis

Ontología ligera para espacios de datos. Alinea IDSA y BIGOWL, enfocándose en la descripción semántica robusta de Activos de Datos y Aplicaciones de Datos a través de Perfiles de Datos, que incluyen características técnicas (DCAT 3) y métricas de calidad granulares. La versión 0.9.0 introduce el concepto de Feature of Interest (alineado con SOSA) para desacoplar sujetos de propiedades, refina la alineación con CRED (UNE 0087:2025) y optimiza el matchmaking semántico.

Introducción volver a índice

EDAAnOWL proporciona un vocabulario compacto pero expresivo para describir la semántica operacional de los espacios de datos y los artefactos analíticos que fluyen a través de ellos. Mientras que el resumen motiva la necesidad, esta introducción clarifica el alcance, las decisiones de diseño y el uso típico. Alcance. EDAAnOWL se enfoca en (i) participantes y roles, (ii) activos de datos y sus perfiles, (iii) ofertas y políticas de uso, y (iv) flujos de trabajo analíticos y aplicaciones. Se alinea con el modelo de información de International Data Spaces (IDSA) para nociones contractuales y de políticas, y se conecta con la familia BIGOWL para describir algoritmos, problemas y flujos de trabajo—vinculando entradas/salidas a través de puertos y perfiles de datos. La ontología fomenta el uso de vocabularios externos curados (AGROVOC, EU Data Theme, DQV) para asegurar anotaciones consistentes e interoperables. Reutilización sobre reinvención. EDAAnOWL reutiliza vocabularios ampliamente adoptados: DCAT para metadatos, SOSA/SSN para observaciones, OWL-Time para tiempo, ODRL/IDSA para políticas, DQV para calidad y PROV-O para procedencia. Estas alineaciones se expresan mediante puentes ligeros para que los activos existentes se integren sin duplicación. Estructura modular. La ontología distingue un núcleo estable de módulos específicos de proyecto. EDAAnOWL adopta una estrategia external-first: prioriza vocabularios normativos internacionales (FAO, UE, OGC), pero permite vocabularios puente locales mínimos cuando aportan alias, acrónimos, alineamientos curados o conceptos ausentes en las fuentes externas. Versionado y persistencia. Cada versión tiene un IRI persistente y versionado. Esto permite la citación inequívoca de una versión concreta ("Esta versión") mientras se preserva un punto de entrada canónico ("Última versión") para facilitar el descubrimiento y la persistencia de enlaces. Uso típico. - Describir un conjunto de datos (DataAsset): 1. Crear una `dcat:Distribution` (o `:DataRepresentation`) y vincularla al DataAsset mediante `ids:representation`. 2. Vincular el perfil a la Distribution usando `edaan:conformsToProfile`. - Definir el perfil: - Clasificar su tipo (p. ej., `edaan:declaresDataClass bigdat:TabularDataSet`). - Definir sus propiedades técnicas (p. ej., `dcat:temporalResolution "P1D"^^xsd:duration`, `edaan:hasCRS `). - Declarar su sujeto de estudio con `edaan:declaresFeatureOfInterest` (p. ej., su objeto es una Parcela de Cultivo). - Listar sus métricas usando `edaan:hasMetric`, describiendo cada `edaan:Metric` con `:metricType` (alineado con `dqv:Metric`), `:metricValue` (valor de la métrica), y `:appliesToFeature` (para métricas por columna). - Publicar una DataApp: Declarar los perfiles que requiere (`edaan:requiresProfile`) y produce (`edaan:producesProfile`), y sus métricas de rendimiento con `:hasPerformanceMetric`. - Habilitar Matchmaking: Permitir que un intermediario encuentre activos que coincidan semánticamente con los requisitos técnicos y de calidad de una aplicación. - Validar Instancias: Usar las formas SHACL proporcionadas, incluyendo el soporte integrado para el Information Model de IDSA (`idsa-shapes.ttl`) y las reglas de EDAAnOWL (`edaan-shapes.ttl`). No-objetivos. EDAAnOWL no es una ontología de dominio monolítica. Proporciona el "tejido conectivo" semántico requerido para publicar, descubrir y componer datos y análisis a través de dominios, delegando la semántica específica a vocabularios autoritativos externos.

EDAAnOWL: Descripción volver a índice

Ontología para espacios de datos que conecta el Modelo de Información IDSA con la familia analítica BIGOWL. Su función principal es describir `DataAssets` (activos) y `DataApps` (servicios) usando `DataProfiles` (perfiles): modelos semánticos que detallan especificaciones técnicas (como resoluciones DCAT 3, CRS) y métricas de calidad granulares (como completitud o unicidad por característica). Esto permite un descubrimiento automatizado robusto y verificación de compatibilidad (matchmaking) para datos y servicios. Además, declara puentes ligeros con DCAT-AP 3.0 (Catalog, DataService, Distribution), ODRL 2.2 (Offer, Rule, Constraint), FOAF (Agent) y metadatos de vocabularios ADMS, siguiendo las directrices del CRED para la anotación de activos de espacios de datos.
